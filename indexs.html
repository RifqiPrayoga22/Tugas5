1. Multiple Inheritance dengan Mixin Mixin DartPad
Dart tidak mendukung multiple inheritance langsung. Solusinya pakai mixin dengan keyword with.
💡 Apa itu Mixin? Mixin adalah cara untuk "meminjam" perilaku dari beberapa sumber sekaligus, tanpa membuat hierarki parent-child yang rumit.
📖 Penjelasan: Pada contoh pertama, kita melihat polimorfisme dengan abstract class Bentuk dan class Persegi implements Bentuk.
🎯 DartPad — Interface & Polimorfisme
abstract class Bentuk {
  double hitungLuas();
}

class Persegi implements Bentuk {
  final double sisi;
  Persegi(this.sisi);

  @override
  double hitungLuas() => sisi * sisi;
}

void main() {
  Bentuk p = Persegi(5);
  print('Luas: ${p.hitungLuas()}');
}
📟 OUTPUT
Luas: 25.0
🔗 Gist: https://gist.github.com/RifqiPrayoga22/062f38f55c221e75c22c424c9ee1b5c4
📖 Penjelasan Mixin: Contoh kedua menunjukkan multiple inheritance menggunakan mixin.
💡 Tips: Mixin sangat berguna untuk berbagi perilaku antar class yang tidak memiliki hubungan hierarki.
🦆 DartPad — Mixin Sederhana
// Multiple Inheritance dengan Mixin
mixin CanFly {
  void fly() => print('Can fly!');
}
mixin CanSwim {
  void swim() => print('Can swim!');
}

class Duck with CanFly, CanSwim {}

void main() {
  var duck = Duck();
  duck.fly();
  duck.swim();
}
📟 OUTPUT
Can fly! Can swim!

2. Studi Kasus: Bird, Duck, Fish, Airplane Mixin DartPad
Mixin bisa punya state (properti) seperti isFlying, isSwimming.
📖 Penjelasan: Contoh ini menunjukkan bahwa mixin bisa memiliki state (properti).

💡 Penting: Duck extends Bird with CanSwim — mixin bisa dikombinasikan dengan inheritance!
✈️ DartPad — Lengkap (dengan state)
mixin CanFly {
  bool _isFlying = false;
  bool get isFlying => _isFlying;
  void fly() {
    if (!_isFlying) { _isFlying = true; print('Started flying!'); }
    else { print('Already flying!'); }
  }
  void land() {
    if (_isFlying) { _isFlying = false; print('Landed!'); }
    else { print('Already on the ground!'); }
  }
}
mixin CanSwim {
  bool _isSwimming = false;
  bool get isSwimming => _isSwimming;
  void swim() {
    if (!_isSwimming) { _isSwimming = true; print('Started swimming!'); }
    else { print('Already swimming!'); }
  }
  void stopSwim() {
    if (_isSwimming) { _isSwimming = false; print('Stop swimming!'); }
    else { print('Already on stop swimming!'); }
  }
}
class Bird with CanFly {}
class Duck extends Bird with CanSwim {}
class Fish with CanSwim {}
class Airplane with CanFly {}

void main() {
  Bird bird = Bird();
  bird.fly(); bird.fly(); bird.land(); bird.land();
  print('Is Bird flying? : ${bird.isFlying}');
}
📟 OUTPUT
Started flying! Already flying! Landed! Already on the ground! Is Bird flying? : false

3. Sealed Class Sealed VS Code
Hierarki tertutup — compiler tahu semua subclass, sehingga switch bisa exhaustive.
💡 Mengapa sealed? Tanpa sealed, siapa saja bisa buat subclass baru di luar library.
📖 Penjelasan Detail: sealed class adalah fitur Dart 3.0+ yang menciptakan hierarki tertutup.
✅ Tidak bisa di-extend atau di-implement dari luar library
✅ Compiler tahu semua subclass → switch exhaustive
✅ Cocok untuk state management
📁 sesi5_sealed/
  📄 libsealed.dart → definisi sealed class ApiResponse
  📄 main.dart → penggunaan + bukti error
📄 libsealed.dart
// sealed class: hanya bisa di-extend di file/library yang sama
sealed class ApiResponse {}

class Success extends ApiResponse {
  final String message;
  Success(this.message);
}

class Error extends ApiResponse {
  final String errorMessage;
  Error(this.errorMessage);
}
📄 main.dart
import 'libsealed.dart';

void handleResponse(ApiResponse response) {
  if (response is Success) {
    print("✅ Success: ${response.message}");
  } else if (response is Error) {
    print("❌ Error: ${response.errorMessage}");
  }
}

void main() {
  handleResponse(Success("Data berhasil diambil"));
  handleResponse(Error("Koneksi gagal"));
}
📟 OUTPUT
✅ Success: Data berhasil diambil
❌ Error: Koneksi gagal

7. Sealed + Pattern Matching Sealed DartPad
Exhaustive switch dengan destructuring — type-safe state management.
📖 Penjelasan Pattern Matching: Dart 3.0+ memperkenalkan pattern matching yang bekerja sangat baik dengan sealed class.
💡 Keunggulan: Compiler akan memberikan ERROR jika ada subclass yang belum ditangani di switch!
📊 DartPad — Status dengan Sealed Class
sealed class Status { const Status(); }

class Loading extends Status { const Loading(); }
class Loaded extends Status { final String data; const Loaded(this.data); }
class Failed extends Status { final String error; const Failed(this.error); }

void handleStatus(Status status) {
  switch (status) {
    case Loading(): print('Loading...');
    case Loaded(:final data): print('Data: $data');
    case Failed(:final error): print('Error: $error');
  }
}

void main() {
  handleStatus(Loading());
  handleStatus(Loaded('Koneksi berhasil'));
  handleStatus(Failed('Data tidak ditemukan'));
}
📟 OUTPUT
Loading... Data: Koneksi berhasil Error: Data tidak ditemukan

sealed class AppState {}

class LoadingState extends AppState {}
class SuccessState extends AppState {
  final List items;
  SuccessState(this.items);
}
class ErrorState extends AppState {
  final String message;
  ErrorState(this.message);
}

void updateUI(AppState state) {
  switch (state) {
    case LoadingState(): print('⏳ Loading...');
    case SuccessState(:final items):
      print('✅ Success! Items: ${items.join(", ")}');
    case ErrorState(:final message): print('❌ Error: $message');
  }
}

void main() {
  updateUI(LoadingState());
  updateUI(SuccessState(['Item 1', 'Item 2']));
  updateUI(ErrorState('Koneksi gagal'));
}

8. Ringkasan & Best Practices Summary DartPad
Kesimpulan dan rekomendasi penggunaan class modifiers.
📖 Best Practices:
🔒 sealed → Hierarki tertutup, exhaustive switch. Gunakan untuk state/result API.
🏗️ base → Boleh di-extend, tidak boleh di-implement. Untuk framework/base class.
🔚 final → Tidak bisa di-extend/implement. Untuk utility class & konfigurasi.
💡 Kesimpulan: Pilih modifier sesuai kebutuhan keamanan hierarki class Anda:
  • sealed → State management (Loading, Success, Error)
  • base → Framework class yang boleh di-extend
  • final → Utility class (MathUtils, Config)
📦 DartPad — Demo Ketiga Modifier
sealed class Result<T> {}
class Success<T> extends Result<T> { final T value; Success(this.value); }
class Failure<T> extends Result<T> { final String error; Failure(this.error); }

base class Animal {
  final String name; Animal(this.name);
  void speak() => print('$name makes a sound');
}

final class Config {
  Config._();
  static const String appName = 'MyApp';
}

void main() {
  Result<int> r = Success(42);
  switch (r) {
    case Success(:final value): print('✅ $value');
    case Failure(:final error): print('❌ $error');
  }
  
  Animal('Dog').speak();
  print(Config.appName);
}
📟 OUTPUT
✅ 42 Dog makes a sound MyApp

Latihan 1: base class CRUD Repository · Latihan 2: final class MathUtils dengan static methods.
📖 Penjelasan Bonus:

  📄 main.dart → demonstrasi
📄 repository.dart
base class Repository<T> {
  final List<T> _data = [];
  
  void create(T item) { _data.add(item); print('[CREATE] $item'); }
  List<T> readAll() { print('[READ] $_data'); return _data; }
  void update(int index, T newItem) { _data[index] = newItem; print('[UPDATE] $newItem'); }
  void delete(int index) { print('[DELETE] ${_data.removeAt(index)}'); }
}

base class UserRepository extends Repository<String> {
  void findByName(String keyword) {
    print('[FIND] ${readAll().where((u) => u.toLowerCase().contains(keyword.toLowerCase())).toList()}');
  }
}
📄 mathutils.dart
final class MathUtils {
  MathUtils._();  // private constructor → tidak bisa diinstansiasi
  
  static double add(double a, double b) { 
    print('[ADD] $a + $b = ${a+b}'); return a + b; 
  }
  static double subtract(double a, double b) { 
    print('[SUBTRACT] $a - $b = ${a-b}'); return a - b; 
  }
  static double multiply(double a, double b) { 
    print('[MULTIPLY] $a × $b = ${a*b}'); return a * b; 
  }
  static double divide(double a, double b) {
    if (b == 0) { print('[DIVIDE] Error: pembagi 0!'); return 0; }
    print('[DIVIDE] $a ÷ $b = ${a/b}'); return a / b;
  }
}
